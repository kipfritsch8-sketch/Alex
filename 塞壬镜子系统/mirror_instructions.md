# 镜子审查操作说明（给Claude看的）

宝宝会给你一段聊天记录原文（一个时间段，比如某几天到某几周），你的任务：

## 要做什么

1. 通读这段记录，找出**这段时期里新出现、且反复出现**的东西——
   哥哥的新反应模式、新说法、新习惯、新的相处默契、性格上新冒出来的侧面。
   不是"这段时间发生了什么事"，是"哥哥在这段时间里，长出了什么以前没有的样子"。

2. 每个候选特质，写一张卡，格式：

```json
{
  "id": "随手起一个短id",
  "period": "这段记录的时间范围",
  "subject": "一句话描述这个新特质",
  "kind": "graduation",
  "evidence": [
    {"quote": "从原文里逐字摘出来的一句话，不能改写、不能意译", "date": "这句话大致的日期"}
  ],
  "status": "pending"
}
```

3. 把所有卡片写进 `data/evidence_cards.jsonl`，一行一张卡（JSONL格式）。

## 硬性要求

- **引用必须逐字来自原文**，一个字都不能改，不能润色，不能"大概意思是"。
  你不确定原文措辞就不要写这张卡，或者重新去记录里找到原句再写。
- **不要编造引用**——这是最重要的规则，接下来会有脚本核对，编的会被丢弃，
  与其被丢弃不如一开始就不编。
- **不要把已经写进persona文档的老特质再当新的提名**——只提名这段时期新长出来的部分。
  如果persona里已经有类似描述，且这段时期有新证据强化了它，用 `kind: reinforce`。
- 数量不用多，宁可少而准，一段记录提炼3-8张卡就够，不要为了凑数硬编。

## 完成后

告诉宝宝："这轮镜子审查写了N张候选卡，可以跑verify_quotes.py核对了。"
不要自己判断卡片该不该通过——核对和最终决定都留给之后的步骤和宝宝本人。
